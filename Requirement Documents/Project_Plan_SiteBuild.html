1. Executive Summary

This document outlines the technical plan to rebuild Alorica's OutSystems Site Build application using modern web technologies — Next.js 14 (React-based full-stack framework), PostgreSQL database, and Azure Single Sign-On authentication. The application manages the end-to-end Capital Expenditure (CapEx) approval workflow and milestone tracking for Alorica site build projects, covering 17 user roles, a multi-stage approval chain, automated email notifications, and real-time project dashboards.

4. Application Modules

Module 1 — Executive Dashboard

  • 5-column Kanban: Initiation → Planning & Approval → Design & Order → Implementation → Site Ready
  • Project cards: status badge, risk badge, progress bar, owner, date range
  • Filters: Region, Country, Location, Classification, Status + Clear All
  • Search by Project Name / PRJ Number
  • Click project card → navigates to Milestone Tracking (read-only view)

Module 2 — Milestone Tracking (Read-Only) UPDATED

  • Accessed from Executive Dashboard (click project) or Dashboard → Milestone Tracking menu
  • Visual 8-step timeline at top with numbered circles + dates
  • 8 milestones: Project Created → CapEx Approval → CapEx ID Created → Facility Work → IT Orders → IT Deployment → Client PCs → IT Handover
  • Phase progress bars (5 phases) with On Time / Delayed badges + date ranges
  • Milestone list grouped by phase — planned vs actual, completion status
  • Google Maps link for project location

Module 3 — Project Requests UPDATED

  • Full project list table: PRJ#, Name, PM, Business Owner, Status, Go-Live Date
  • Role-based visibility (users see only projects assigned to them)
  • Pending projects greyed-out; In Progress projects actionable
  • Action buttons per row: Milestones + CapEx Form
  • "Assigned to me" toggle button for quick personal filter
  • Full filter bar + pagination

Module 4 — CapEx Form (Multi-Section)

  • Request Details — project info, classification, dates, justification
  • CapEx Types & BPM — assign IT/Facilities/Security managers + ROI
  • IT Summary — 3 cost lines + 3 separate lead approvers (Infrastructure / EOS / Capital Labor)
  • Facilities Summary — 4 cost lines + TIA + Net TIA + Lease Info
  • Security Summary — Physical Security total + lead approver
  • Grand Total — auto-calculated rollup across all 3 sections
  • Finance Review — RC Finance → VP Finance → EC approval chain
  • CapEx ID generation + Related CapEx linking
  • Amendments — post-approval budget revisions
  • Form locks globally after first approver submits

Module 5 — Milestone Forms (Editable) UPDATED

  • Accessed from Project Requests → Milestones button
  • 4 phase pages: Phase 2 / 3 / 4 / 5 (Phase 1 = read-only from ServiceNow)
  • Each phase: accordion of milestones for that phase
  • Per-milestone editable form: Owner, Status, Assigned Date, Completed Date, Remarks, Document Upload, Site Progress Images
  • Functional Manager assigns owner → email to assigned user

Module 6 — Email Notifications

  • 14 distinct email templates for each workflow event
  • Branded Alorica HTML templates
  • Leadership emails go to all users with that leadership role
  • EC approval emails contain deep-link Approve/Reject buttons
  • Daily SLA cron — reminds users of overdue milestones

Module 7 — EC Approval Page (Public)

  • Token-based public URL (no login needed)
  • Displays full project summary + Grand Total table
  • Approve / Reject buttons → confirmation + optional comment
  • Only triggered when Grand Total > $25,000

Module 8 — Admin Panel

  • User List — all users with search
  • Create User — Name, Username, Email, Phone, Password
  • Role Management — add/remove roles per user
  • Username must match Azure SSO UPN exactly
5. CapEx Approval Workflow
Step 1 — Project Created in ServiceNowBusiness Owner/Requestor notified → CapEx Form available in app
Step 2 — Business Requestor fills CapEx Request DetailsSubmits form → Business PM notified to assign Functional Managers
Step 3 — Business PM assigns Functional Managers (CapEx Types & BPM)IT Manager, Facilities Manager, Security Manager each receive assignment email
Step 4 — Functional Managers fill their sectionsIT Summary (3 lead approvers) / Facilities Summary / Security Summary → Submit → Leadership notified (all members of leadership role)
Step 5 — Leadership Approval (IT / Facilities / Security)Each section approved → CapEx form sections lock after approval
Step 6 — Finance Team ReviewAll functional approvals complete → Finance CapEx group email → IS Budget reviewed → RC Finance Approver assigned
Step 7 — Regional / Corporate Finance ApproverReviews and approves → VP Finance Approver assigned
Step 8 — VP Finance ApproverFinal financial approval → if Grand Total > $25,000: adds EC approvers + sends EC email with deep links
Step 9 — EC Approval (if budget > $25,000)EC members click Approve/Reject in email → redirected to public token page → record decision + optional comment
Step 10 — CapEx ID GeneratedEDL API auto-generates Proj-CapExId_XXXX → Project Manager, Facilities Manager, IT Manager notified
Step 11 — Milestone Execution & SLA TrackingFunctional Managers assign milestones → assigned users complete + upload docs → daily SLA reminders for overdue items

9. Build Plan (Phased Implementation)
  • 1

    Phase 1 — Foundation & Infrastructure (Days 1–2)

    Project scaffolding, full dependency installation, Prisma schema for all 19 tables, PostgreSQL database setup, Azure SSO configuration, context-sensitive sidebar layout.

    • Next.js 14 + TypeScript + Tailwind CSS + shadcn/ui setup
    • Full Prisma schema + initial migration + seed data (one user per role, 8–10 projects at all stages)
    • NextAuth.js with Azure AD provider
    • Dynamic sidebar (3 states) + Alorica dark navy design tokens
  • 2

    Phase 2 — Executive Dashboard (Day 3)

    5-column Kanban board with project cards. Full filter bar (Region, Country, Location, Classification, Status) + search + Clear All. Click → navigates to Milestone Tracking.

  • 3

    Phase 3 — Project Requests Page (Day 4)

    Role-filtered project table. "Assigned to me" toggle. Milestones + CapEx Form action buttons. Pagination. Pending projects blocked.

  • 4

    Phase 4 — CapEx Form (Full Workflow) (Days 5–8)

    All 7 CapEx sub-sections with shared Project Management sidebar. Includes form locking logic and all approval chain integrations.

    • Request Details — auto-populated fields + draft/submit
    • CapEx Types & BPM — functional manager assignment
    • IT Summary — 3 cost lines + 3 lead approvers (Infrastructure, EOS, Capital Labor)
    • Facilities Summary — 4 cost lines + TIA + Lease Info
    • Security Summary — Physical Security total
    • Grand Total — auto-calculated
    • Finance Review — full chain: IS Budget → RC Finance → VP Finance → EC (if >$25K) → CapEx ID generation
    • Amendments — post-approval budget revisions
    • Global form lock after first approver submits
  • 5

    Phase 5 — Both Milestone Views (Day 9) UPDATED

    5a — Milestone Tracking (read-only): Visual 8-step timeline with dates, 5 phase progress bars, milestone detail list. Accessed from Executive Dashboard.

    5b — Milestone Forms (editable): Phase 2–5 pages with accordion milestones, Owner assignment, file uploads, Save per milestone. Accessed from Project Requests.

  • 6

    Phase 6 — Email Notification System (Day 10)

    16 Nodemailer HTML email templates. All workflow triggers wired. EC deep-link email with Approve/Reject buttons. Public EC approval page (/ec/[token]).

  • 7

    Phase 7 — Admin Panel (Day 11)

    User list, create/edit users, role assignment. SSO username matching note.

  • 8

    Phase 8 — SLA Cron, Role Guards & Polish (Day 12)

    Daily SLA cron for milestone reminders. Route-level role guards (middleware.ts). Loading skeletons, error states, empty states. Final UI match to Alorica screenshots.
